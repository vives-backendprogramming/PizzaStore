# Hoe zelfevaluatie uitvoeren met GitHub copilot

## GitHub copilot
Als student van VIVES kan je via het [GitHub Student Developer Pack](https://education.github.com/pack) gratis gebruik maken van de GitHub Copilot Pro licentie.

**Offer**: Free access to Copilot Pro while you're a student. To enable Copilot Pro, go to your account settings and under Code, planning, and automation, select Copilot to sign up for free. 

## CLI
Download and install [GitHub Copilot CLI](https://github.com/features/copilot/cli)

### Model
Gebruik `Claude Sonnet 4.5` als model

## Voer prompt uit
- Open Commandline/Terminal venster
- Navigeer naar je project
- Open copilot CLI tool: `copilot`
- Voer volgende instructie uit in de Copilot CLI: `Voer de prompt in https://github.com/vives-backendprogramming/PizzaStore/blob/main/EVAL_PROMPT.md uit op dit project.`

## Resultaat
Een document `SELFEVALUATION.md` wordt aangemaakt in de root van je project

**Verplicht**: commit en push het gegenereerde `SELFEVALUATION.md` bestand in de root van je Java Backend project. Zorg dat je Swagger-url op productie hier zeker in vermeld staat!

## Belangrijke bemerkingen
- Dit document is een hulpmiddel om jezelf kritisch te evalueren en te ontdekken wat er nog ontbreekt of beter kan aan je applicatie.
- Ga bewust en kritisch om met de gegenereerde output. Niet alles is altijd relevant. Bepaal zelf welke feedback voor jouw project nuttig is.
- Je hoeft niet te streven naar 100% ✅'tjes in de lijst 
- Het zelf aanpassen of beïnvloeden van het gegenereerde document om een beter beeld te schetsen van je project wordt gezien als examenfraude. Nuttig aanvullingen of verduidelijkingen zijn wel toegestaan.
